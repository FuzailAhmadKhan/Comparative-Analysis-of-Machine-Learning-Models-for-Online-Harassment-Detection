Proposed Framework Against Online Harassment

Step 1: Data Collection & Account Monitoring

The system continuously monitors social media platforms, collecting comments and interactions from potentially problematic accounts. It identifies accounts that repeatedly engage with hateful content through likes, shares, and comments.

Data Collection Metrics:
  • Comment history analysis
  • Interaction patterns (likes, shares)
  • Network mapping of related accounts
  • Frequency of flagged content engagement
Step 2: Machine Learning Classification

Using the trained models (SVM with Bigram TF-IDF performing best), the system classifies content into three categories:

Hate Speech

Content containing targeted harassment or threats

Offensive Language

Contains offensive terms but not targeted harassment

Neutral Content

No offensive language or hate speech detected

Step 3: Network Analysis & Chain Detection

The system identifies networks of accounts that coordinate harassment campaigns by analyzing interaction patterns and content similarity.

Step 4: Moderator Alert & Action

When harmful networks are detected, the system alerts human moderators with detailed reports and recommended actions based on platform policies.

Moderator Dashboard Features:
  • Priority-ranked case management
  • Detailed interaction history
  • Network visualization tools
  • Automated action recommendations
  • Appeal handling workflow
  • Performance analytics

About This Research

This system is based on research comparing machine learning classifiers for detecting targeted harassment on social media platforms. The study evaluated three feature engineering techniques and five machine learning algorithms using a dataset of 14,509 annotated tweets.

Key Findings:
  • Bigram features with TF-IDF outperformed unigram approaches
  • Random Forest and SVM achieved the highest accuracy (90.33% and 90.44% respectively)
  • The proposed framework enables pseudo-autonomous detection of coordinated harassment campaigns
Dataset Information:

The Hate Speech Detection dataset contains tweets classified into three categories:

  • HATE: Contains hate speech (2,399 texts)
  • OFFENSIVE: Contains offensive language but no hate speech (4,836 texts)
  • OK: No offensive content (7,274 texts)
Note: This demonstration uses a simulated classification system. In a production environment, the models would be trained on the actual dataset and integrated with social media platform APIs.
